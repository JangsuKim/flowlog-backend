# 📘 Flowlog - 요구사항 정의서
[🇯🇵 日本語版はこちら](./01_Requirements_ja.md)

## 1. 개요
**Flowlog**은 팀과 개인의 작업 흐름을 한눈에 파악하고,  
업무의 생산성을 향상시키기 위한 **칸반(Kanban) 스타일의 프로젝트 관리 도구**입니다.  
이 문서는 초기 릴리스 버전에 대한 요구사항 정의를 정리한 것입니다.

---

## 2. 목적
- 작업 진행 상황을 **시각적으로 파악**할 수 있게 한다.  
- 팀원 간의 **업무 공유와 상태 관리**를 효율적으로 지원한다.  
- **가벼운 구조와 확장성**을 갖춘 현대적인 웹 아키텍처를 적용한다.

---

## 3. 주요 사용자
| 구분 | 설명 |
|------|------|
| **개인 사용자** | 본인의 할 일을 체계적으로 정리하고 시각화하고 싶은 사람 |
| **팀 리더** | 여러 팀원의 작업 상황을 한눈에 파악하고 싶은 사람 |
| **중소기업 / 스타트업** | JIRA나 Asana처럼 복잡한 시스템까지는 필요하지 않지만, 기본적인 프로젝트 관리 도구가 필요한 조직 |

---

## 4. 시스템 범위
| 항목 | 내용 |
|------|------|
| **지원 환경** | PC 웹 브라우저 (Chrome, Safari, Edge) |
| **모바일 지원** | 스마트폰 앱은 미지원 (향후 고려) |
| **지원 언어** | 한국어, 일본어, 영어 |
| **대상 규모** | 개인 ~ 50명 규모의 팀 사용을 목표로 함 |

---

## 5. 기능 요구사항

### 5.1 사용자 관리
- 회원가입 / 로그인 / 로그아웃  
- JWT 토큰 기반 인증 및 인가  
- 사용자 프로필 수정 (이름, 아이콘 등)

### 5.2 프로젝트 관리
- 프로젝트 생성 / 수정 / 삭제  
- 프로젝트별 참여 멤버 지정  
- 진행 상태(활성 / 완료 / 보관) 관리  

### 5.3 작업(Task) 관리
- 작업 생성 / 수정 / 삭제  
- 작업 상태(Todo / Doing / Done) 변경 (드래그 앤 드롭 지원)  
- 담당자, 우선순위, 마감일 설정  

### 5.4 대시보드
- 전체 프로젝트의 진행 상태를 차트로 시각화  
- 완료율, 지연 작업, 우선순위별 통계 표시  
- 주간 및 월간 단위의 분석 뷰 제공  

---

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|------------|
| **성능** | 일반적인 작업 조작에 대해 1초 이내 응답 |
| **확장성** | 최대 1,000개의 작업(Task)까지 안정적으로 처리 가능 |
| **보안성** | JWT 인증 + HTTPS 통신 필수 |
| **가용성** | Docker 기반 24시간 365일 구동 가능 구조 |
| **유지보수성** | 모듈화된 구조로 기능 확장이 용이해야 함 |

---

## 7. 제약 조건
- 데이터베이스는 **MySQL**을 사용한다.  
- 개발 및 배포 환경은 **Docker Compose** 기반으로 구성한다.  
- 프론트엔드는 **React (Next.js 14)**, 백엔드는 **Spring Boot 3**을 사용한다.  
- 초기 배포 환경은 **Vercel(Frontend)** + **Render(Backend)**로 설정한다.

---

## 8. 향후 개선 계획
- 댓글 / 파일 첨부 기능 추가  
- 팀 단위 실시간 협업 (WebSocket 기반) 지원

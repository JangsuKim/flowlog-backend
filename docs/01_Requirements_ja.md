# 📘 Flowlog - Requirements Definition Document
[🇰🇷 한국어 버전 보기](./01_Requirements_ko.md)

## 1. Overview
**Flowlog** は、チームおよび個人のタスクを視覚的に管理し、生産性を向上させることを目的とした **カンバン方式のプロジェクト管理ツール** です。  
本ドキュメントは、初期リリースにおける要件定義をまとめたものです。

---

## 2. Purpose
- 作業の進捗をリアルタイムで可視化する。
- チームメンバー間のタスク共有とステータス管理を容易にする。
- 軽量でモダンなアーキテクチャにより、拡張性を確保する。

---

## 3. Target Users
| タイプ | 詳細 |
|--------|------|
| **個人ユーザー** | 自身のタスクを整理・可視化したい人。 |
| **チームリーダー** | 複数メンバーの進行状況を把握したい人。 |
| **中小企業** | JIRAやAsanaのような大規模ツールまでは不要だが、基本的なタスク管理を行いたい組織。 |

---

## 4. System Scope
| 項目 | 説明 |
|------|------|
| **対応デバイス** | PCブラウザ（Chrome, Safari, Edge） |
| **非対応** | スマートフォンアプリ（将来的に検討） |
| **対応言語** | 日本語・英語・韓国語 |
| **ユーザー数** | 1～50人規模を想定 |

---

## 5. Functional Requirements

### 5.1 ユーザー管理
- サインアップ／ログイン／ログアウト
- JWTトークンによる認証・認可
- プロフィール編集（ユーザー名・アイコン）

### 5.2 プロジェクト管理
- プロジェクトの新規作成／編集／削除
- 参加メンバーの割り当て
- ステータスごとの進行管理

### 5.3 タスク管理
- タスクの作成／編集／削除
- ステータス（Todo, Doing, Done）の変更（ドラッグ＆ドロップ対応）
- 担当者、優先度、期限日設定

### 5.4 ダッシュボード
- プロジェクト全体の進捗グラフ表示
- 完了タスク率、遅延タスク表示
- 週単位・月単位の統計

---

## 6. Non-Functional Requirements

| 区分 | 要件 |
|------|------|
| **パフォーマンス** | 1秒以内のレスポンス（標準タスク操作） |
| **スケーラビリティ** | 小規模チーム向け（最大1000タスク） |
| **セキュリティ** | JWT + HTTPS通信必須 |
| **可用性** | 24時間365日稼働を想定（Docker構成） |
| **拡張性** | モジュール設計により新機能を容易に追加可能 |

---

## 7. Constraints
- データベースは MySQL を使用。
- 開発環境は Docker 上で構築。
- フロントエンドは React (Next.js 14)、バックエンドは Spring Boot 3。
- インフラは Vercel（Front）+ Render（Back）を利用。

---

## 8. Future Enhancements
- コメント・添付ファイル機能の追加。
- チームリアルタイムコラボレーション（WebSocket）対応。
- 通知機能（Slack連携など）。
